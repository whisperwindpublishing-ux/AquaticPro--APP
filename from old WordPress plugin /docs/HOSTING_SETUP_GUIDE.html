AquaticPro Hosting & Deployment Guide

Stack: Hetzner VPS  +  SpinupWP  +  MainWP   |   Date: March 2026

Summary: Each client gets a fully isolated WordPress install on a shared VPS. SpinupWP manages server config, SSL, and Nginx automatically. MainWP lets you push plugin updates to all sites in one click.

1. Why This Architecture

This Setup (Separate Installs)WordPress Multisite
User data isolation✅ Fully isolated⚠️ Shared user table
Content isolation✅ Separate databases✅ Separate table prefixes
Plugin updates✅ MainWP one-click✅ One install
One bad update affects others✅ Isolated⚠️ All sites affected
SSL management✅ Auto via SpinupWP✅ Auto
Per-client customization✅ Flexible⚠️ Constrained

Key reason to avoid Multisite: WordPress Multisite shares one global wp_users table. Client A's employees would exist network-wide — unacceptable for HR/employee data.

3. Initial Server Setup

Step 1 — Create Hetzner VPS

  1. Go to console.hetzner.cloud → New Project → Add Server
  2. Choose CX22 (Ubuntu 22.04 LTS)
  3. Add your SSH public key
  4. Note the server's public IP address

Step 2 — Connect to SpinupWP

  1. Sign up at spinupwp.com
  2. Dashboard → Add Server → "I'll provide my own server"
  3. Enter your Hetzner IP and root SSH credentials
  4. SpinupWP provisions Nginx, PHP-FPM, MySQL, Let's Encrypt automatically (~10 min)
You never need to touch the terminal. SpinupWP handles all server configuration, SSL renewals, and Nginx vhosts going forward.

4. One-Time: Build the Template Site

Create a master template that gets cloned for every new client.

Step 1 — Create the template site in SpinupWP

  1. SpinupWP → Add Site → Domain: template.aquaticpro.org
  2. SSL provisions automatically
  3. Install WordPress via SpinupWP one-click

Step 2 — Configure WordPress

  1. Log into WordPress admin
  2. Delete default post, page, and comment
  3. Delete unused themes (keep Twenty-Twenty-Four or similar)
  4. Settings → General: set timezone, disable anyone can register
  5. Settings → Reading: discourage search engines (optional for template)
  6. Settings → Discussion: disable comments globally
  7. Settings → Permalinks: set to Post name

Step 3 — Install Plugins

  1. Upload and activate aquaticpro.zip via Plugins → Add New → Upload
  2. Install and activate MailPoet from the WordPress plugin directory
  3. Configure MailPoet: From name, From email, sending method (use MailPoet's sending service or your SMTP)
  4. Install and activate MainWP Child plugin (needed for MainWP management later)

Step 4 — Snapshot the template

  1. In SpinupWP → Site → Backups → Run backup now
  2. Label it "AquaticPro Template v1"
  3. This becomes the source for all future client clones

5. Per-Client: New Site Workflow

Estimated time per new client: ~5–10 minutes

Step 1 — DNS

  1. In your DNS provider (wherever aquaticpro.org is managed), add an A record:
Type: A
Name: clientname
Value: [your Hetzner IP]
TTL: 3600

Step 2 — Create site in SpinupWP

  1. SpinupWP → Add Site → Domain: clientname.aquaticpro.org
  2. Under "Clone from existing site" → select your template site
  3. SpinupWP clones all files and the database
  4. SSL certificate issues automatically

Step 3 — Update site URL (post-clone)

  1. Log into the new site's WordPress admin
  2. Settings → General → verify WordPress Address and Site Address show clientname.aquaticpro.org (SpinupWP usually handles this automatically — just verify)

Step 4 — Connect to MainWP

  1. On the new site, go to Plugins → MainWP Child → copy the Security Key
  2. In your MainWP Dashboard site → Add New Site → enter URL + security key
  3. Site is now managed centrally
New client site is live with: SSL ✅   AquaticPro plugin ✅   MailPoet configured ✅   Clean WordPress ✅

6. Setting Up MainWP Dashboard

MainWP Dashboard runs on its own WordPress install — a private management site on the same VPS.

  1. In SpinupWP → Add Site → Domain: manage.aquaticpro.org (keep this private/password protected)
  2. Install WordPress → Install MainWP Dashboard plugin
  3. Follow the MainWP setup wizard
  4. Add each client site using their MainWP Child security keys

7. Pushing Plugin Updates to All Clients

When you build a new aquaticpro.zip:

  1. Go to your MainWP Dashboard site
  2. MainWP → Plugins → Upload Plugin → upload aquaticpro.zip
  3. Select all client sites
  4. Click Update / Activate

All client sites receive the update simultaneously. No manual SSH, no individual logins.

9. Backups

SpinupWP includes scheduled backups. Configure in SpinupWP → Server → Backups:

  • Storage: Backblaze B2 (cheapest, ~$0.006/GB/mo) or AWS S3
  • Schedule: Daily database backup, weekly full backup
  • Retention: 30 days recommended

10. Quick Reference Checklist

New client onboarding checklist:

  • ☐ Add DNS A record for clientname.aquaticpro.org
  • ☐ SpinupWP → Add Site → Clone from template
  • ☐ Verify Site URL in WordPress Settings → General
  • ☐ Connect to MainWP Dashboard
  • ☐ Create admin user for client
  • ☐ Enable SpinupWP backups for new site
  • ☐ Test AquaticPro plugin loads correctly
  • ☐ Test MailPoet sends a test email

When shipping a plugin update:

  • ☐ Run npm run zip in VS Code
  • ☐ Test on template site first
  • ☐ MainWP → Upload Plugin → push to all sites
